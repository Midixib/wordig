# GitHub → Vercel デプロイ クイックチェックリスト

初めての人向けの「やることだけ」一覧です。  
詳しい説明は **「デプロイ手順_GitHubからVercel.md」** を見てください。

---

## 事前準備

- [ ] GitHubアカウントを作った（https://github.com/signup）
- [ ] Vercelアカウントを作った（https://vercel.com/signup）※GitHubでログイン推奨
- [ ] Gitをインストールした（https://git-scm.com/download/win）
- [ ] ターミナルで `git --version` が動くことを確認した

---

## パート1：GitHubにコードを上げる

- [ ] GitHubで「New repository」を作った（READMEは追加しない）
- [ ] Cursorでプロジェクトフォルダを開き、ターミナルを開いた
- [ ] `git init` を実行した
- [ ] `git add .` を実行した
- [ ] `git commit -m "初回：サイトをGitHubに上げる"` を実行した
- [ ] `git remote add origin https://github.com/自分のユーザー名/リポジトリ名.git` を実行した
- [ ] `git branch -M main` を実行した
- [ ] `git push -u origin main` を実行した
- [ ] GitHubのページを更新して、ファイルが表示されることを確認した

---

## パート2：Vercelでサイトを公開する

- [ ] Vercelに「Continue with GitHub」でログインした
- [ ] 「Add New…」→「Project」をクリックした
- [ ] 作ったGitHubリポジトリを選んで「Import」した
- [ ] 設定はそのままで「Deploy」をクリックした
- [ ] ビルドが終わるまで1〜2分待った
- [ ] 表示されたURLをクリックして、サイトが表示されることを確認した

---

## 2回目以降：更新して再公開するとき

- [ ] ファイルを編集して保存した
- [ ] ターミナルで `git add .` を実行した
- [ ] `git commit -m "〇〇を変更した"` を実行した
- [ ] `git push` を実行した
- [ ] 1〜2分後に同じURLで新しい内容が表示されることを確認した

---

## つまったとき

| 症状 | 見る場所 |
|------|----------|
| pushでパスワードが通らない | 手順書「よくあるつまずき」→ Personal Access Token |
| Vercelのビルドが失敗する | 手順書「よくあるつまずき」→ ビルド失敗 |
| 変更がサイトに反映されない | 手順書「よくあるつまずき」→ サイトが更新されない |

詳細は **「デプロイ手順_GitHubからVercel.md」** を開いて、該当の見出しを読んでください。
