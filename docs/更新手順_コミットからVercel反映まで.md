# 更新手順：新しいコミット → GitHub → Vercel反映

※ 初回デプロイは済んでいる前提です。

---

## 1. 新しい名前（メッセージ）でコミットして GitHub に送る

### 手順

1. **Cursor** でプロジェクト（例：wordig）を開く。
2. ファイルを編集して **保存** する。
3. **ターミナル** を開き、プロジェクトのフォルダにいることを確認する。

   プロジェクトフォルダにいない場合：
   ```bash
   cd C:\Users\ichioka mizuki\wordig
   ```

4. 次の3つを **順番に** 実行する。

   ```bash
   git add .
   ```
   → 変更したファイルをすべて「次のコミットに含める」と指定する。

   ```bash
   git commit -m "ここに変更内容を短く書く"
   ```
   → 「新しい名前（メッセージ）でのコミット」はこの **-m "〇〇"** の部分。  
   例：`"トップの文言を修正"` や `"ボタンの色を変更"` など。

   ```bash
   git push
   ```
   → そのコミットを GitHub に送る。

これで「新しい名前（メッセージ）付きのコミット」が GitHub に反映されています。

---

## 2. Vercel に反映される流れ

- **Vercel は、GitHub のリポジトリと連携している場合、`git push` を検知して自動で再デプロイします。**
- 特別な操作は不要です。

### 確認のしかた

1. **Vercel のダッシュボード**（https://vercel.com/dashboard）を開く。
2. 該当プロジェクトをクリックする。
3. **「Deployments」** タブで、いちばん上に **新しいデプロイ** が並び、状態が **Building → Ready** になるのを待つ（だいたい 1〜2 分）。
4. 状態が **Ready** になったら、そのプロジェクトの **URL**（例：`https://〇〇.vercel.app`）にアクセスすると、最新のコミット内容が反映されています。

---

## まとめ

| やりたいこと | やること |
|--------------|----------|
| 新しい名前でコミットして GitHub に送る | `git add .` → `git commit -m "メッセージ"` → `git push` |
| それを Vercel に反映させる | 何もしない（push で自動再デプロイ） |
| 反映されたか確認する | Vercel ダッシュボードの Deployments で Ready になるまで待つ → URL で表示確認 |

**「新しい名前」＝ コミットメッセージ（`-m "〇〇"`）** です。ここを変えると、GitHub の履歴に「何をしたコミットか」が残り、Vercel のデプロイ一覧でもどのコミットでビルドしたか分かります。
